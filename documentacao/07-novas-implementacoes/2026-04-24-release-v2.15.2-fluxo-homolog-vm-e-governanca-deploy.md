# Release v2.15.2 - fluxo homolog-vm e governanca de deploy

Data: `24/04/2026`

## Objetivo

Formalizar o fluxo oficial de quatro ambientes do ERP para reduzir risco operacional entre desenvolvimento, homologacao e producao.

## O que mudou

- a branch `develop-desktop` passa a ser a trilha oficial de desenvolvimento do `PC` e do `notebook`;
- a branch `homolog-vm` passa a concentrar tudo que sera validado na `VM Ubuntu 24`;
- a branch `main` passa a receber apenas o que foi homologado na `VM`;
- a `VPS` continua consumindo somente a `main`, sempre com backup previo de codigo, banco e arquivos.

## Fluxo oficial consolidado

```text
PC / notebook -> GitHub (develop-desktop) -> GitHub (homolog-vm) -> VM Ubuntu 24 -> GitHub (main) -> VPS
```

## Regras operacionais formalizadas

1. desenvolvimento diario acontece em `develop-desktop`;
2. promocao para homologacao acontece por merge em `homolog-vm`;
3. a `VM` testa exclusivamente `homolog-vm`;
4. promocao para producao acontece da `VM` para `main`;
5. deploy da `VPS` exige backup Git, backup do banco e backup dos arquivos antes de `git pull --ff-only origin main`.
