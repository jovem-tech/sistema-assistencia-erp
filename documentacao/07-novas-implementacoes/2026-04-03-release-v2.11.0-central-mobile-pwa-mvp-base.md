# Release v2.11.0 - Central Mobile PWA (MVP paralelo)

Data: 03/04/2026  
Versao: `2.11.0`  
Escopo: `Mobile/PWA + API interna` (sem substituir a Central web)

## Objetivo

Entregar a fundacao de um aplicativo mobile/PWA oficial, integrado ao mesmo banco e backend atual do ERP, com evolucao progressiva e baixo retrabalho.

## Implementacao

1. API interna mobile v1 no CodeIgniter 4 (`/api/v1`) com autenticacao por Bearer token.
2. Endpoints operacionais para auth, usuarios, clientes, OS, conversas, mensagens, notificacoes e push subscriptions.
3. Stream SSE (`/api/v1/realtime/stream`) para tempo real com fallback de polling incremental no app.
4. Migration complementar de banco para infraestrutura mobile:
   - `mobile_api_tokens`
   - `mobile_push_subscriptions`
   - `mobile_notifications`
   - `mobile_notification_targets`
   - `mobile_event_outbox`
5. Controller de entrada `/atendimento-mobile` com bloqueio de desktop (override por `?preview=1`) e destino configuravel por `mobile_pwa_url`.
6. Scaffold do app paralelo em `mobile-app/` (Next.js + Service Worker + manifest + telas de conversas, OS e notificacoes).
7. Suporte de publicacao em subrota (`NEXT_PUBLIC_APP_BASE_PATH`), com proxy Nginx em `/atendimento-mobile-app` e processo PM2 dedicado (`assistencia-mobile-pwa`).
8. Rewrite server-side do Next para API (`/api/v1`) sem depender de CORS no MVP.
9. Manifest PWA reforcado com icones PNG (`192`, `512`, `maskable`) e metadata para compatibilidade de instalacao mobile.
10. Mapeamento de ajuda/documentacao atualizado para `openDocPage('atendimento-mobile')`.
11. Fluxo de instalacao assistida no login do PWA com card dedicado para:
   - acionar `beforeinstallprompt` por botao quando elegivel;
   - exibir instrucoes manuais em iOS e navegadores sem banner automatico;
   - sinalizar bloqueio por falta de HTTPS valido.
12. Hardening do fluxo de push no mobile:
   - diagnostico de prontidao no modulo `Notificacoes` (suporte, HTTPS, standalone iOS, permissao, VAPID);
   - bloqueio preventivo do botao quando pre-requisitos nao estao atendidos;
   - mensagens de erro especificas para facilitar ativacao em iPhone.
13. Diagnostico expandido de push iOS:
   - validacao de versao minima (iOS 16.4+);
   - exibicao dos status de `Service Worker`, `PushManager` e `Notification API`;
   - botao de teste local para confirmar permissao e exibicao de notificacao no dispositivo.
14. Hotfix de publicacao por subdominio:
   - ajuste de `NEXT_PUBLIC_ERP_WEB_BASE_URL` para dominio HTTPS do ERP;
   - correcoes no install do `sw.js` para cache resiliente sem falha global em `cache.addAll`.
