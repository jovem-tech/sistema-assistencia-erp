# Release v2.2.14 - Timeout configuravel e aviso de sessao expirada

Data: 25/03/2026

## Objetivo
Eliminar expiracoes imprevisiveis de login por divergencia de timeout e avisar de forma clara quando a sessao realmente expirar, antes de o usuario perder tempo tentando salvar formularios.

## O que mudou

### 1. Timeout de inatividade configuravel
Arquivos principais:
- `app/Views/configuracoes/index.php`
- `app/Controllers/Configuracoes.php`
- `app/Helpers/sistema_helper.php`

Mudanca:
- criada a aba `Sessao e Seguranca` em `Configuracoes`
- novo campo `sessao_inatividade_minutos`
- sanitizacao entre `5` e `1440` minutos

Impacto:
- o administrador passa a controlar o tempo de inatividade sem editar codigo;
- o sistema deixa de depender de timeout fixo hardcoded.

### 2. Backend com fonte unica de expiracao
Arquivos principais:
- `app/Filters/AuthFilter.php`
- `app/Config/Session.php`
- `app/Controllers/Sessao.php`
- `app/Config/Routes.php`

Mudanca:
- `AuthFilter` passa a usar o timeout configurado como unica regra de inatividade;
- respostas AJAX/fetch agora retornam `401` JSON com `session_expired`, `message` e `redirect_url`;
- criado `GET /sessao/heartbeat`;
- a sessao tecnica do CodeIgniter e ajustada para nao expirar antes da regra operacional de inatividade.

Impacto:
- expiracao de sessao fica previsivel;
- o frontend consegue identificar o motivo da falha e orientar o usuario corretamente.

### 3. Monitor global de sessao no frontend
Arquivos principais:
- `app/Views/layouts/main.php`
- `app/Views/layouts/embed.php`
- `public/assets/js/scripts.js`

Mudanca:
- layouts protegidos publicam meta tags com timeout, heartbeat e login;
- frontend monitora clique, digitacao, foco, toque, scroll e movimento;
- heartbeat discreto so ocorre quando ha atividade real;
- ao expirar, o sistema usa SweetAlert2 e redireciona o contexto correto para o login.

Impacto:
- reduz o risco de formularios longos expirarem silenciosamente;
- melhora o fluxo em modais embed e telas com AJAX.

## Validacao tecnica executada
- `php -l app/Helpers/sistema_helper.php`
- `php -l app/Config/Session.php`
- `php -l app/Filters/AuthFilter.php`
- `php -l app/Controllers/Sessao.php`
- `php -l app/Controllers/Configuracoes.php`
- `php -l app/Views/configuracoes/index.php`
- `php -l app/Views/layouts/main.php`
- `php -l app/Views/layouts/embed.php`
- `node --check public/assets/js/scripts.js`
- `php spark routes` confirmando `GET /sessao/heartbeat`

## Observacao
Nao houve migracao de banco. A persistencia usa a tabela `configuracoes` ja existente, adicionando apenas uma nova chave logica.
