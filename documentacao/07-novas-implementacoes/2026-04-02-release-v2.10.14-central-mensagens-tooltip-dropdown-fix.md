# Release v2.10.14 - Central de Mensagens: hotfix tooltip/dropdown + timeout de gateway

Data: 02/04/2026  
Versao: `2.10.14`

## Objetivo

Eliminar warnings de conflito Bootstrap na barra de acoes da conversa e reduzir falhas de envio por timeout curto do provider local.

## Problemas observados

- Console com erro repetido:
  - `Bootstrap doesn't allow more than one instance per element. Bound instance: bs.tooltip.`
- Envio em alguns cenarios com retorno `503 Service Unavailable` em picos de latencia.

## Implementacao

### 1) Tooltip x Dropdown (Frontend)

- A inicializacao de tooltip da Central passou a ignorar elementos cujo `data-bs-toggle` nao seja `tooltip`.
- Botoes da action bar que sao dropdown (`modo de atendimento` e `mais acoes`) ficaram fora da inicializacao Bootstrap Tooltip.
- Quando existir tooltip legado nesses toggles, a instancia e descartada defensivamente.

Arquivo:
- `public/assets/js/central-mensagens.js`

### 2) Timeout padrao de gateway local (Backend)

- Timeout default aumentado de `20s` para `30s` nos providers locais:
  - `api_whats_local` (Windows/dev)
  - `api_whats_linux` (Linux/VPS)

Arquivos:
- `app/Services/MensageriaService.php`
- `app/Services/WhatsApp/LocalGatewayProvider.php`

## Resultado esperado

- Sem spam de erro Bootstrap no console ao abrir/usar dropdowns da action bar.
- Menor incidencia de `503` por timeout em ambientes com latencia elevada.
- Quando houver indisponibilidade real do gateway, o `503` permanece como resposta correta para observabilidade operacional.
