# Release v2.11.3 - Mobile PWA SW Response Hardening

Data: 03/04/2026

## Objetivo

Eliminar o erro de Service Worker:

- `Uncaught (in promise) TypeError: Failed to convert value to 'Response'`.

## Causa raiz

No handler de `fetch` do arquivo `mobile-app/public/sw.js`, quando a rede falhava e nao havia item no cache, o fluxo retornava `undefined` para `event.respondWith`, quebrando o contrato esperado pelo navegador.

## Implementacao

- `CACHE_NAME` atualizado para `assistencia-mobile-v3`.
- `fetch` do SW reescrito para fluxo `async/await` com retorno garantido de `Response`.
- fallback offline adicionado:
  - retorna cache quando existir;
  - em navegacao sem cache, retorna `Response` `503` com texto `Offline`.
- requests nao HTTP agora sao ignoradas no SW (`req.url.startsWith("http")`).
- falhas no `cache.put` passaram a registrar erro tecnico sem interromper resposta da requisicao.

## Arquivo alterado

- `mobile-app/public/sw.js`

## Resultado esperado

- Service Worker deixa de lançar `Failed to convert value to 'Response'`.
- PWA fica mais resiliente em perda de conectividade e em primeira carga sem cache pronto.
