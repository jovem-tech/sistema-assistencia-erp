# Release v2.11.1 - Push mobile inbound com envio real

Data: 03/04/2026  
Versao: `2.11.1`  
Escopo: `Central de Mensagens + API mobile/PWA`

## Objetivo

Corrigir o problema de notificacao mobile nao entregue mesmo com permissao habilitada no dispositivo.

## Implementacao

1. Adicionado envio Web Push real no backend com VAPID via dependencia `minishlink/web-push`.
2. Criado `app/Services/Mobile/WebPushService.php` para:
   - montar payload padrao (`title`, `body`, `route`);
   - enviar para subscriptions ativas;
   - registrar erro tecnico em falha de entrega;
   - desativar subscriptions expiradas (`ativo = 0`).
3. `MobileNotificationService` foi atualizado para disparar push automaticamente apos `createNotification`.
4. `CentralMensagensService` foi atualizado para criar notificacao mobile ao receber inbound de cliente:
   - evento `message.inbound`;
   - titulo com remetente;
   - preview textual/midia;
   - rota `/conversas/{id}`.
5. Mantida compatibilidade com o fluxo atual:
   - sem alterar a UX da Central web;
   - sem duplicar tabelas do ERP;
   - sem quebrar API existente.

## Configuracao obrigatoria em producao

ERP (`.env`):

```env
MOBILE_VAPID_PUBLIC_KEY=
MOBILE_VAPID_PRIVATE_KEY=
MOBILE_VAPID_SUBJECT=mailto:suporte@jovemtech.eco.br
```

Mobile app (`mobile-app/.env.production`):

```env
NEXT_PUBLIC_VAPID_PUBLIC_KEY=
```

Observacao: a chave publica do app deve ser a mesma do backend.
