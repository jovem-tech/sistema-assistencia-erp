# Release v2.11.2 - Mobile PWA com abertura completa de OS

Data: 03/04/2026

## Objetivo

Concluir no PWA mobile o fluxo de abertura de OS com paridade funcional ao cadastro de abertura do ERP web, mantendo o mesmo backend CodeIgniter 4 e o mesmo banco de dados.

## Entregas

### 1) API de metadados para a abertura

- Novo endpoint `GET /api/v1/orders/meta`.
- Retorna dados necessarios para montar a tela de abertura no mobile:
  - clientes;
  - equipamentos por cliente;
  - tecnicos ativos;
  - status e prioridades;
  - defeitos por tipo de equipamento.

### 2) Criacao completa de OS na API mobile

- `POST /api/v1/orders` ampliado para receber os campos da abertura completa:
  - operacionais (`status`, `prioridade`, `tecnico`, datas);
  - tecnicos (`diagnostico`, `solucao`, observacoes);
  - comerciais/financeiros (`valores`, `desconto`, `pagamento`, `garantia`);
  - colecoes (`defeitos`, `acessorios_data`, `estado_fisico_data`);
  - fotos de entrada via `multipart/form-data`.
- Compatibilidade reforcada para upload em ambos formatos de campo:
  - `fotos_entrada`
  - `fotos_entrada[]`

### 3) Frontend PWA (Next.js)

- Nova tela `mobile-app/src/app/os/nova/page.tsx` com formulario completo.
- Botao `Nova OS` adicionado na listagem de OS mobile (`/os`).
- Envio via `FormData` com anexos de fotos de entrada.
- Estilos adicionados para os novos componentes do formulario (secao, grids, chips e lista de defeitos).

## Resultado esperado

- Operador mobile consegue abrir OS completa no PWA sem depender da tela web do ERP.
- Fluxo permanece integrado ao banco existente, sem duplicacao de entidades.
- Estrutura pronta para evolucoes futuras de produtividade mobile.
