# Release v2.7.1 - Ajuste real do banco `erp` e limpeza controlada da base atual

Data: 28/03/2026  
Versao: `2.7.1`

## O que entrou nesta release

### 1. Ajuste do importador para o schema real do legado
- a conexao legada local passou a apontar para o banco `erp`
- a query de `clientes` foi adaptada para a estrutura real da tabela `clientes`
- como o legado nao possui um cadastro de equipamentos por cliente compativel com o ERP novo, a etapa `equipamentos` passou a derivar um snapshot por OS a partir da tabela `os`
- o `legacy_id` do equipamento derivado ficou padronizado como `os-{id_legado}`

### 2. Preflight real validado
- o `legacy:preflight` passou a concluir sem erros bloqueantes no banco `erp`
- totais encontrados:
  - `1295` clientes
  - `3560` equipamentos derivados
  - `3560` OS
- clientes sem telefone valido passaram a gerar aviso, nao bloqueio

### 3. Limpeza controlada da base atual antes da carga real
Foi adicionada uma etapa formal para apagar os dados ficticios do sistema atual antes da importacao real:

```bash
php spark legacy:prepare-target
php spark legacy:prepare-target --execute
php spark legacy:import --execute --wipe-target
```

Escopo:
- clientes
- equipamentos
- OS
- tabelas operacionais diretamente relacionadas
- uploads de fotos/documentos ligados a OS e equipamentos

Preservado:
- usuarios
- permissoes
- configuracoes
- catalogos estruturais

## Motivacao

A migracao legada saiu do modo "estrutura generica" e passou a refletir o banco antigo real em operacao, reduzindo risco na virada e garantindo uma etapa segura para limpar os dados ficticios da homologacao antes de carregar os dados reais.
